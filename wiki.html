Code Convention

  1. boolean

    • boolean은 is/has 등 be동사로 시작해야 한다.

  2. 들여쓰기 : 탭 vs 띄어쓰기

    • 들여쓰기는 띄어쓰기 2칸으로 통일한다.

    • 필요시 에디터에서 탭 키를 띄어쓰기 2칸으로 수정해서 사용한다.

  3. 주석 : // vs /** */

    • // : 1줄 주석 작성시에만 사용한다.

    • /** */ : 2줄 이상 주석은 반드시 해당 주석을 사용한다.

  4. 중복 : 클래스 내부 메서드, 변수 등

    • 중복 클래스는 생략한다.

      ex)

      X)

      class Car {

      boolean isCarExtend;

      }

      O)

      class Car {

      boolean isExtend;

      }

    • 기타 중복 규칙은 FE/BE 별로 정한다.

  5. 기타

    • 클래스명, 변수, 메서드, 반복문, 조건문 등은 프론트 / 백엔드끼리 논의

    Commit Convention

    1. feat : 새로운 기능 추가

      • git commit -m "feat: 회원가입 시 이메일 인증 기능 추가"

        새로운 사용자가 등록할 때, 이메일을 통한 인증 단계를 추가함. 이제 사용자는 등록 후 이메일 링크를 클릭하여 계정을 활성화할 수 있음.

    2. fix : 버그 수정

      • git commit -m "fix: 시작 시 응용 프로그램 충돌 버그 해결"

        응용 프로그램이 시작될 때 발생하는 특정 상황에서 발생하는 충돌 버그를 해결. 이제 응용 프로그램은 정상적으로 시작됨.

    3. docs : 문서 수정

      • git commit -m "docs: 새로운 API 엔드포인트에 대한 상세 문서 업데이트"

        새로운 API 엔드포인트의 기능, 파라미터 및 응답에 대한 상세한 문서를 추가. 개발자들은 이를 통해 새로운 기능을 더 쉽게 적용할 수 있음.

    4. style : 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우

      • git commit -m "style: main.js 파일의 코드 스타일 일관성 수정"

        main.js 파일의 코드 스타일을 Prettier를 사용하여 일관성 있게 수정. 들여쓰기 및 세미콜론이 올바르게 적용.

    5. refactor : 코드 리펙토링

      • git commit -m "refactor: 사용자 유효성 검사를 위한 코드 리펙토링"

        사용자 유효성을 검사하는 부분을 단순화하고 중복 코드를 제거하여 유지보수성을 향상시킴.

    6. test : 테스트 코드, 리펙토링 테스트 코드 추가

      • git commit -m "test: 사용자 인증에 대한 새로운 통합 테스트 추가"

        보안 강화 및 안정성 향상을 위해 사용자 인증 부분에 대한 새로운 통합 테스트를 추가.

    7. chore : 빌드 업무 수정, 패키지 매니저 수정

      • git commit -m "chore: 프로덕션 배포를 위한 빌드 스크립트 업데이트"

        프로덕션 배포 시 필요한 빌드 스크립트를 최적화하여 배포 프로세스를 더욱 신속하고 안정적으로 만듦.
